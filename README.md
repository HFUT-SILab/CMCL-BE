# Collaborative Multi-Cue Learning with Explicit Bifurcation Encoding for Open-Set Palm Vein Recognition

This repository contains the minimal runnable implementation of **CMCL-BE**, a palm vein recognition method built around collaborative multi-cue learning and explicit bifurcation-aware encoding. The released code focuses on the DRAF TransMixer mainline used for open-set palm vein verification.

> Paper status: manuscript in preparation. BibTeX will be added after publication information is finalized.

## Highlights

- Multi-cue palm vein representation with texture, directional, and global context branches.
- DRAF-enhanced Mamba/TransMixer context modeling.
- Boundary attention refinement for discriminative vein features.
- Open-set evaluation pipeline with AUC, EER, and TAR at FAR thresholds.
- VMamba/selective_scan source is vendored for reproducible setup.

## Environment

The reference server environment used:

- Python 3.10
- PyTorch `2.5.1+cu124`
- torchvision `0.20.1+cu124`
- CUDA 12.4
- GPU training/evaluation is expected

For CUDA-enabled PyTorch, install PyTorch from the official CUDA wheel index first, then install the remaining dependencies.

### WSL/Linux Installation

```bash
cd /mnt/d/lyj/code/pyproject/CMCL-BE

conda create -n cmcl-be python=3.10 -y
conda activate cmcl-be

pip install torch==2.5.1 torchvision==0.20.1 --index-url https://download.pytorch.org/whl/cu124
```

## Dataset Format

The training root should use one subdirectory per identity:

```text
train_2/
|-- identity_0001/
|   |-- image_001.png
|   `-- image_002.png
|-- identity_0002/
|   `-- image_001.png
`-- ...
```

The validation root should contain dataset-specific folders:

```text
val/
|-- CASIA/
|-- HFUT/
|-- PolyU/
|-- TongJi/
`-- VERA/
```

Images are resized internally to `128 x 128`. Directional features are generated online by `triditional_method/EBOCV.py`.

Datasets and pretrained checkpoints are not included in this repository. Please prepare them locally and pass paths through command-line arguments.

## Training

Run from WSL/Linux:

```bash
cd /mnt/d/lyj/code/pyproject/CMCL-BE
export PYTHONPATH=$PWD:$PYTHONPATH

python train_val/train.py \
  --train-root /mnt/d/lyj/dataset/vein/train_2 \
  --output ./output \
  --batch-size 16 \
  --num-workers 4 \
  --draf-tau 1.0 \
  --draf-sparse-lambda 0.001
```

Checkpoints are saved under:

```text
output/all_data/
```

## Validation

```bash
python train_val/val.py \
  --val-root /mnt/d/lyj/dataset/vein/val \
  --output ./output \
  --train-split all \
  --datasets CASIA HFUT PolyU TongJi VERA \
  --batch-size 16
```

Validation writes per-dataset results under `output/<DATASET>/`, including result text files and ROC data generated by the metric script.

## One-Command Pipeline

```bash
python train_val/auto.py \
  --train-root /mnt/d/lyj/dataset/vein/train_2 \
  --val-root /mnt/d/lyj/dataset/vein/val \
  --output ./output \
  --batch-size 16 \
  --num-workers 4
```

To summarize the best validation results only:

```bash
python train_val/show_best.py --output ./output --train-split all
```
